## §四十四 subagent 返回内容前端隐藏 system prompt

**Phase**: 56
**背景**: nanobot 核心 Phase 42 已在 subagent 返回消息中插入 `<!-- nanobot:system -->` 标记（在引导 prompt 之前）。前端需要识别此标记并截断显示。

**改动**:
- `frontend/src/pages/chat/MessageItem.tsx` — 渲染消息内容时，检测 `<!-- nanobot:system -->` 标记
  - 如果存在，只显示标记之前的内容
  - 标记及其后面的引导 prompt 隐藏不显示
  - 纯渲染层处理，不改变消息存储
  - 不对旧数据做兼容截断（只处理包含标记的新消息）

**验收**:
1. 包含 `<!-- nanobot:system -->` 标记的消息，前端只显示标记前的内容
2. 不包含标记的旧消息正常显示
3. 复制消息时也只复制可见部分

## §四十五 SSE 刷新保持用户浏览位置

**Phase**: 56
**背景**: 前端 SSE 推送消息时，强制滚动到底部，打断用户浏览历史消息。

**改动**:
- `frontend/src/pages/chat/MessageList.tsx` — 修改滚动逻辑：
  - **用户发送消息** → 总是滚动到底部
  - **SSE 流式推送** → 根据当前滚动位置判断：
    - 用户在底部附近（阈值 ~150px）→ 自动跟随滚动到底部
    - 用户在上方浏览历史 → 保持当前位置不动

**验收**:
1. SSE 推送时，用户在底部 → 自动跟随
2. SSE 推送时，用户在上方 → 位置不变
3. 用户发送消息 → 始终滚到底

## §四十六 /session 命令补充 cache 信息 + web 端支持

**Phase**: 56
**背景**: gateway 侧 `/session` 命令只显示基础 token 信息，缺少 cache 字段；web-chat 前端没有 `/session` 命令。

**改动 1 — gateway `/session` 补充 cache 信息**:
- 文件: `nanobot/agent/loop.py`
- 在 `/session` 处理逻辑中，token_line 增加 cache 信息
- 格式：`缓存命中: 12.3K · 缓存写入: 5.6K · 未缓存: 3.2K`
- 数据来源：`get_session_usage()` 返回 cache_creation_input_tokens 和 cache_read_input_tokens
- 同时更新 `nanobot/usage/recorder.py` 的 `get_session_usage()` 方法添加 cache 字段

**改动 2 — web-chat 前端支持 `/session` 命令**:
- 文件: `frontend/src/store/messageStore.ts`
- 在 slash 命令拦截中新增 `/session`
- 显示内容与 gateway 侧对齐：session key、状态、usage（含 cache 信息）
- 数据通过已有的 API 获取

**验收**:
1. gateway `/session` 显示 cache_creation 和 cache_read 信息
2. web-chat 前端支持 `/session` 命令
3. 两端格式和内容基本对齐

## §四十七 web-subsession 父子关系注册

**Phase**: 56
**背景**: batch-orchestrator 通过 web-subsession (curl) 启动的 Worker session 无法被前端识别父子关系，因为启发式规则在混合调用链下失效。

**改动 1 — webserver 新增单条追加 API**:
- 文件: `webserver.py`
- 新增 `POST /api/sessions/parents` 支持单条追加
- 请求体：`{"child": "child_session_id", "parent": "parent_session_id"}`
- 实现：读取 session_parents.json → 追加 → 写回（文件锁避免并发冲突）

**改动 2 — create_subsession.sh 新增 --parent 参数**:
- 文件: `skills/web-subsession/scripts/create_subsession.sh`
- 新增 `--parent <session_id>` 可选参数
- 创建 session 后，如果指定了 --parent，curl POST 注册父子关系

**改动 3 — Skill 文档更新**:
- `skills/web-subsession/SKILL.md` — 说明 `--parent` 参数用法
- `skills/batch-orchestrator/SKILL.md` — Worker Prompt 模板中加入 `--parent` 用法说明

**验收**:
1. `POST /api/sessions/parents` API 正常工作
2. `create_subsession.sh --parent xxx` 能自动注册父子关系
3. 前端能通过 parentMap 正确展示父子关系

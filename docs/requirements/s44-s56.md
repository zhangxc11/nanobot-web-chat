## §四十四 subagent 返回内容前端隐藏 system prompt

**Phase**: 56
**背景**: nanobot 核心 Phase 42 已在 subagent 返回消息中插入 `<!-- nanobot:system -->` 标记（在引导 prompt 之前）。前端需要识别此标记并截断显示。

**改动**:
- `frontend/src/pages/chat/MessageItem.tsx` — 渲染消息内容时，检测 `<!-- nanobot:system -->` 标记
  - 如果存在，只显示标记之前的内容
  - 标记及其后面的引导 prompt 隐藏不显示
  - 纯渲染层处理，不改变消息存储
  - 不对旧数据做兼容截断（只处理包含标记的新消息）

**验收**:
1. 包含 `<!-- nanobot:system -->` 标记的消息，前端只显示标记前的内容
2. 不包含标记的旧消息正常显示
3. 复制消息时也只复制可见部分

## §四十五 SSE 刷新保持用户浏览位置

**Phase**: 56
**背景**: 前端 SSE 推送消息时，强制滚动到底部，打断用户浏览历史消息。

**改动**:
- `frontend/src/pages/chat/MessageList.tsx` — 修改滚动逻辑：
  - **用户发送消息** → 总是滚动到底部
  - **SSE 流式推送** → 根据当前滚动位置判断：
    - 用户在底部附近（阈值 ~150px）→ 自动跟随滚动到底部

**验收**:
1. SSE 推送时，用户在底部 → 自动跟随
2. SSE 推送时，用户在上方 → 位置不变
3. 用户发送消息 → 始终滚到底

**改动 3 — Skill 文档更新**:
- `skills/web-subsession/SKILL.md` — 说明 `--parent` 参数用法
- `skills/batch-orchestrator/SKILL.md` — Worker Prompt 模板中加入 `--parent` 用法说明

**验收**:
1. `POST /api/sessions/parents` API 正常工作
2. `create_subsession.sh --parent xxx` 能自动注册父子关系
3. 前端能通过 parentMap 正确展示父子关系

## §四十九 Subagent 可见性 — 前端运行标识

**验收**:
1. 运行中 session 显示绿色脉冲点
2. 新 session 出现时自动刷新列表
3. Subagent 进度信息实时更新
4. 无运行 session 时不产生多余请求

## §五十 Bug 修复 — Session 列表轮询闪烁

**Phase**: 57 Hotfix
**背景**: §四十八~§四十九 引入的轮询机制导致 session 列表每 10 秒闪烁一次。

**根因分析**:

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 1 | `useRunningSessions` 每次轮询无条件 `setRunningKeys(new Set())`，即使内容未变也创建新引用 | `useRunningSessions.ts` | SessionList 每 10s 全部重渲染 |
| 2 | `useSubagentStatus` 每次轮询无条件 `setStatusMap(new Map())`，同理 | `useSubagentStatus.ts` | SessionList 每 5s 全部重渲染 |
| 3 | `useRunningSessions` 变化时调 `fetchSessions()` → `sessionStore` 无条件设 `loading: true` → `Sidebar.tsx` 根据 loading 隐藏整个 SessionList → 重新加载后显示 | `sessionStore.ts` + `Sidebar.tsx` | 列表先消失再出现 = 闪烁 |

**修复方案**:

1. **`useRunningSessions.ts`**: 将 `setRunningKeys()` 移到 changed 判断内部，仅在内容真正变化时才更新 state
2. **`useSubagentStatus.ts`**: 新增 `mapsEqual()` 深比较函数（JSON 序列化对比），仅在内容变化时 setState
3. **`sessionStore.ts`**: `fetchSessions()` 区分首次加载（设 loading）和后台刷新（静默更新，不设 loading）
4. **`Sidebar.tsx`**: 移除 `loading` 门控，SessionList 始终渲染（数据更新由 zustand 驱动，无需隐藏/显示切换）

**额外修复 — 前端未重新构建**:
- 66cd02a 提交修改了源码但未执行 `npm run build`，`frontend/dist/` 仍为旧版本
- 需重新 `npm run build` + 重启 webserver 才能生效

**验收**:
1. session 列表在任务执行期间不再闪烁
2. 运行中标识（绿点）和 subagent 进度仍正常显示
3. 新 session 出现时列表仍能自动刷新

## §五十一 前端隐藏标记改为闭合标签 + 仅 user 消息生效

**Phase**: 58
**背景**: 原有 `<!-- nanobot:system -->` 标记为单标签截断模式，现改为闭合标签对 `<!-- nanobot:system -->...<!-- /nanobot:system -->`，并限制仅对 user role 消息生效。

**改动 1 — 常量更新**:
- 新增 `SYSTEM_MARKER_OPEN` 和 `SYSTEM_MARKER_CLOSE` 常量
- 移除旧的 `SYSTEM_MARKER` 单常量

**改动 2 — stripSystemMarker 算法重写**:
- 有开始+有结束 → 移除标签对之间内容，保留前后可见文本
- 有开始+无结束 → fallback 截断开始标签及之后（向后兼容）
- 无开始 → 原样返回

**改动 3 — getTextContent 增加 role 过滤**:
- 新增 `role` 参数，仅 `user` role 调用 `stripSystemMarker()`
- assistant/tool/system role 的消息不做 strip
- 所有调用处传入 `message.role`

**验收**:
1. user 消息中闭合标签对内容被正确隐藏
2. 无闭合标签时 fallback 截断（向后兼容）
3. assistant 消息中含 `<!-- nanobot:system -->` 文本时不被截断

## §五十二 Turn 结束后「滚动到底部」提示按钮

**Phase**: 58
**背景**: 执行过程中用户可能在查看上方内容，turn 结束后需要提示用户有新消息。

**需求**:
- turn 结束后：如果用户不在底部，右下角出现「↓ 新消息」提示按钮
- 点击按钮：平滑滚动到底部，按钮消失
- 用户已在底部时不显示
- 用户手动滚到底部后按钮也消失

**实现**:
- 在 MessageList 组件中添加滚动检测逻辑
- 新建 ScrollToBottomButton 内联组件
- 使用 CSS transition/animation 做出现/消失效果
- 监听 `sending` 从 true → false 的变化作为 turn 结束事件

**验收**:
1. turn 结束时若不在底部，按钮出现
2. 点击按钮平滑滚到底部，按钮消失
3. 已在底部时不显示
4. 用户手动滚到底部后按钮消失
